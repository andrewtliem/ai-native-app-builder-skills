# Advanced Bundle

**Purpose:** Optional structure for larger projects, feature folders, GitHub issues, and richer checklists.

**When to use:** Use for capstone-level work or stronger teams.

## Included Skills / Practices

| Skill | Role |
|---|---|
| `feature folder pattern` | Use docs/ai-native/features/001-feature-name/ for larger apps. |
| `quality checklists` | Generate privacy, security, UX, demo, or grading checklists. |
| `github issue export` | Convert implementation issues into GitHub issues when appropriate. |
| `multi-agent planning` | Split work safely across agents only after artifacts are stable. |

## Expected Student Artifacts

- `docs/ai-native/features/001-feature-name/spec.md`
- `docs/ai-native/features/001-feature-name/plan.md`
- `docs/ai-native/features/001-feature-name/tasks.md`
- `docs/ai-native/features/001-feature-name/evidence.md`

## Teaching Note

Students should run these skills one at a time, read the saved artifact, make a human decision, revise if needed, then continue. The bundle is a teaching path, not an autopilot.

## Repository Paths

Installable skills live under `skills/<skill-name>/SKILL.md`. This bundle is only a learning path; it does not duplicate the skill content.
