# Spec-Driven Bundle

**Purpose:** Make the PRD buildable before any AI coding starts.

**When to use:** Use before coding. This is the main Spec Kit-inspired addition.

## Included Skills / Practices

| Skill | Role |
|---|---|
| `14-project-constitution` | Define non-negotiable project principles and quality gates. |
| `15-clarify-prd` | Ask up to five important questions to remove ambiguity. |
| `04-prd-to-architecture` | Turn the PRD into technical architecture and design decisions. |
| `05-architecture-to-issues` | Break architecture into small implementation issues. |
| `16-analyze-artifacts` | Check PRD, constitution, architecture, and issues for gaps and contradictions. |

## Expected Student Artifacts

- `docs/ai-native/14-project-constitution.md`
- `docs/ai-native/15-clarification-log.md`
- `docs/ai-native/04-architecture.md`
- `docs/ai-native/05-issues.md`
- `docs/ai-native/16-artifact-analysis.md`

## Teaching Note

Students should run these skills one at a time, read the saved artifact, make a human decision, revise if needed, then continue. The bundle is a teaching path, not an autopilot.
