# Simple Todo App Example

This example shows how a student project can use the AI-Native App Builder Skill Pack from raw idea to proof.

The app is intentionally small: a local-first todo app for students who want to track course tasks.

## Teaching Use

Use this as a reference when students ask, “What should my saved artifacts look like?”

This is not meant to be perfect production documentation. It is a compact classroom example showing:

- requirement IDs (`FR-###`)
- user story IDs (`US-###`)
- success criteria IDs (`SC-###`)
- small implementation issues (`T###`)
- traceability from intent to evidence
- honest limitations and postmortem
